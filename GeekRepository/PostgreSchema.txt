# Platform auth tables (Dapper-managed; applied via Migrations/Sql/*.sql)

Core tables used by GeekBackend today:

- users, user_secrets, user_claims
- devices_oauth (device registration and challenge-response; not OAuth protocol storage)
- two_factor_pending_sessions, two_factor_trusted_devices
- sync_queue, sync_conflicts, websocket_sessions
- audit_log, security_incidents, device_reregistration_requests
- rbac_* (roles, permissions) as present in database

Removed by migration 0004_drop_oauth_openiddict_and_adapter_storage.sql:

- openiddict_applications, openiddict_scopes, openiddict_authorizations, openiddict_tokens
- jti_blacklist, oidc_storage
- geek_auth.OAuthClient, geek_auth.OAuthToken, geek_auth.OidcStorage
- auth.oauth_clients, auth.oauth_client_states
- schema geek_seo (product schema; see Geek-SEO repo)

Content tables (EF Core / AppDbContext): departments, case_studies, use_cases, etc.
